# Task Completion Checklist

- [ ] Code follows DRY, SOLID, KISS, and LEAN principles.
- [ ] No new global variables introduced.
- [ ] Functions and modules are kept small and focused.
- [ ] Magic numbers are moved to `constants.h` or appropriate headers.
- [ ] `make clean && make` succeeds without warnings.
- [ ] Visual verification of the keyboard on screen (if possible/applicable).
- [ ] Verify that labels update correctly (Shift/Caps Lock) if touched.
- [ ] Ensure `_NET_WM_WINDOW_TYPE_DOCK` is correctly set for Always on Top behavior.
